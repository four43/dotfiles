## How Bots See Your Page

When a human visits a web page, their browser downloads HTML, runs JavaScript,
loads CSS, renders animations, and presents the final interactive result. Bots,
crawlers, and AI agents don't always get that full experience. Here's what
different consumers actually see:

| Consumer | What it fetches | JS executed? | Sees animated values? |
|---|---|---|---|
| **Googlebot** | HTML, then renders JS (delayed) | Yes (eventually) | Sometimes — depends on timing |
| **Other search engines** (Bing, etc.) | Mostly HTML | Limited | Rarely |
| **curl / wget** | Raw HTML only | No | No — sees initial DOM values |
| **LLM agents** (ChatGPT, Claude, etc.) | Usually rendered text via reader mode | Varies | Usually yes if SSR, no if client-only |
| **Screen readers** | Live DOM (post-JS) | Yes | Depends on timing — may read "0" before animation completes |
| **Social media previews** | `<meta>` OG tags only | No | N/A — uses meta tags, not page content |
| **RSS readers** | Feed XML or fetched HTML | No | No |

**Key takeaway:** If important content (statistics, prices, data) only reaches
its correct value after JavaScript animation, many consumers will see the wrong
value. Server-render the final values in HTML and use CSS/JS for visual
animation only.
